KGDB setup

Kernel debugging over serial needs three pieces: a UART reachable from outside, kgdboc pointed at it, and a debugger on the host speaking the GDB remote protocol. On the Port log device the console UART is only exposed on the debug connector.

## Kernel configuration

Enable the debugger and keep the watchdog from biting while execution is halted — see CFI violations.

CONFIG_KGDB=y CONFIG_KGDB_SERIAL_CONSOLE=y kgdboc=ttySAC0,115200 kgdbwait
A halted kernel tells the truth; a printk tells a story.

Notes for implementation. Fonts: EB Garamond (display: note titles, book names, panel headings), Spectral (note prose), Alegreya Sans (all chrome), JetBrains Mono (source mode, code, counts, kbd). Tree indentation is depth × 13px with 1px indent guides; active row uses --active-pill plus --text-bright. Wiki-links use --link with a 50%-opacity solid underline; unresolved links use --text-soft with a dashed underline. A block in source mode sits on --edit-bg in mono. The stitched dotted line 16px from the page's left edge is the single decorative element on the page — keep it. Binding chips are per-book data, not theme tokens; they stay identical across themes.

Action layer. Sidebar toolbar (26px hit targets, 15px stroke icons at 1.8 weight, hover = --active-pill): new note, new section, search, collapse-all, hide sidebar. New note lands in the currently selected section (or book root); creating means writing Untitled.md and immediately opening block 0 in source mode so typing the title is the first act. Tab strip above the page: the active tab uses --bg with a --sidebar-line border so it fuses with the editor area; inactive tabs are borderless --text-soft; × (or middle-click) closes; ⌘T opens a tab, ⌘W closes. Tabs are UI state (QML-side), not vault state. Sync pill in the topbar: accent dot = synced, --text-dim dot = offline, pulsing = syncing; clicking it opens sync details. Backend hooks these imply: Library.create_note(parent_id), Library.create_section(parent_id), and a sync-status signal once the sync engine exists.

Star map (sightlines). Sits at the top of the Marginalia pane on --code-bg. The current note is the center star: --link ring around a --text-bright core. Incoming references are --text-soft dots, outgoing links are --link dots, unresolved links are hollow --text-dim dots on dashed sightlines; lines are 0.7px --text-dim at ~55% opacity, labels 7px mono uppercase. Layout is a plain radial spread — angle by index with slight radius jitter, no force simulation. Clicking a dot opens that note (unresolved dots create it — same link_unresolved hook as in the editor). Data is what the pane already has: Backlinks.for_note() plus the current note's outgoing links, so the QML side is one Canvas/Shape component with no new backend beyond an outgoing_links() slot on NoteEditor. Cap at ~10 dots and let the card list below carry the overflow.

Vault picker. Shown when Booklet starts without a vault argument. Centered column on --sidebar: mark, name, version, one primary button (--accent fill, --sidebar text — the only filled button in the app), then two cards on --panel. Card 1: Zuletzt geöffnet — vault name in the display face, path in mono --text-dim, relative time right-aligned, × removes from the list only (never touches the vault on disk); the dot is the vault's own accent, stored in the recents entry. Card 2: the three actions — create, open folder, connect to sync server (the third ties into the sync base feature: server URL + device token, then pulls the remote vault). Recents persist as JSON in the platform config dir (QStandardPaths::AppConfigLocation equivalent — dirs::config_dir()/booklet/recent.json), most recent first, capped at 8. Backend surface: recent_vaults(), remove_recent(path), create_vault(path); opening reuses the existing Library.open_vault, which also bumps the recents entry.
